description: Не читать чекбоксы как истину — сверять с журналом и контекстом разговоров перед показом статуса
type: feedback

Не выдавать статус целей по голым чекбоксам в файлах. Перед дейликом — сначала прочитать журнал за последние дни, понять что реально произошло, обновить файлы целей, и только потом показывать статус пользователю.

**Why:** пользователь несколько раз поймал на ошибках — "0% целей" когда реально 2 из 5 закрыты, "перенос 4 дня" на задачу которая не планировалась, "монетизация не решена" когда решение уже принято. Это раздражает и подрывает доверие.

**How to apply:** в начале daily-review, после загрузки данных — сначала сверить journal/ записи с состоянием целей в monthly/weekly/daily.md. Обновить чекбоксы на основе фактов. Только потом формировать картину для пользователя. Если не уверен — лучше спросить, чем утверждать.

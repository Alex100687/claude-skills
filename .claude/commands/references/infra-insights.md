# Инсайты, советы и кейсы

Практические советы из реального опыта работы с AI-инфраструктурой. Используй эту информацию чтобы давать более точные рекомендации.

## Технические инсайты

### Три слоя памяти Claude Code
1. **CLAUDE.md** — автозагрузка при каждом старте
2. **MEMORY.md** — авто-память до 200 строк (по инструкции)
3. **MCP memory** — persistent граф знаний без лимита
Каждая сессия начинается с полным контекстом.

### Скилл = папка, не файл
Частая ошибка: создают `~/.claude/skills/hello.md` вместо `~/.claude/skills/hello/SKILL.md`. Скилл — это всегда **папка** с файлом SKILL.md внутри.

### Перенос контекста между Claude
Попросить существующего Claude «дай контекст про меня» и перенести в серверный Claude. Мгновенная настройка нового окружения без ручного объяснения.

### VPN не нужен для Claude на сервере
Сервер в Америке → Claude ходит в Anthropic из Америки. VPN нужен только вашему ноутбуку, не серверу.

### Codex: «выполняй незамедлительно»
Без этого Codex любит принять задачу и ничего не делать. В AGENTS.md или в промпте добавлять: «выполняй и отчитайся».

### Ultrathink
Недокументированная команда Claude Code для extended thinking. Жрёт токены, но для сложных задач незаменима. Использовать осознанно.

### Золотое правило: Exa Search перед задачей
Всегда делать Exa Search перед началом новой задачи. AI не знает всего, но может собрать актуальную информацию из интернета. Особенно важно для библиотек, API, документации.

### Claude = CTO, Codex = инженер
Claude для глубокого анализа и архитектуры. Codex для быстрого кода. Идеальная пара — разные модели, разные сильные стороны. Через мультиагентные системы можно заставить их работать вместе.

### Почему CLI, а не Cursor
CLI даёт больше контроля, работает по SSH из любого места, память персистентна, можно автоматизировать через хуки и скиллы. Claude Code в терминале мощнее чем в GUI-приложении.

### tmux vs systemd — простое правило
- **tmux** — для интерактивной работы: Claude Code, Codex, ручные команды
- **systemd** — для сервисов 24/7: боты, веб-серверы, мониторинг
Если работаете руками → tmux. Если процесс без вас → systemd.

## Кейсы участников

### MTProxy через Claude
Участник попросил Claude поднять MTProxy для Telegram прямо на VPS. Поднял, проверил — работает. Полезно для обхода блокировок Telegram.

### Speech-to-text через Claude
Вместо Upload2Server — нативная интеграция с Telegram через channels. Claude сам настроил speech-to-text по запросу.

### Полная инфраструктура за 5 часов
За один воркшоп: Claude Code + Codex + VPN на роутер + MTProxy для Telegram.

### HPE Configurator (3 дня)
Обработка 8000 PDF-документов, декомпиляция C#-приложения, RAG-система, SQLite-база, AI-помощник на GPT-4o. Создано за 3 дня с помощью Claude Code.

### Murmur — мультиагентная система
3 сервера (2 США, 1 Англия). Claude = техдир, Codex пишет код, Haiku проверяет. Шифрованная связь между агентами.

## Частые вопросы

### Max vs Pro подписка
Только лимиты. На Pro недельный лимит достигается при активной работе. Max = безлимит для серьёзной работы.

### Если выключить VPN на компе, Claude продолжит?
Да! Сервер работает 24/7. VPN нужен только ноутбуку. Claude на сервере не зависит от VPN.

### Если РКН заблокирует IP сервера
Купить второй IP у хостера, Claude сам перенастроит всё за минуты.

## Production cron — реальные примеры
```bash
# Очистка кешей (npm, uv, claude debug) — ежедневно в 4:00
0 4 * * * rm -rf /root/.cache/uv/* /root/.npm/_npx /root/.local/share/pnpm/store/v3/tmp

# Публикация дайджеста — 3 раза в день
0 1,5,9 * * * cd /opt/lifecoach && node scripts/clawdigest_tweet.cjs

# Автопостинг в Telegram — каждые 5 минут
*/5 * * * * /opt/lifecoach/sys-adm-bot/venv/bin/python poster.py

# Ротация бэкапов — воскресенье в 4:30
30 4 * * 0 ls -t /root/.claude-mem/backups/*.db* | tail -n +6 | xargs -r rm

# Мониторинг диска >80%
0 */6 * * * df / | awk 'NR==2 && $5+0 > 80 {print "DISK WARNING: "$5" used"}' >> /var/log/disk-alert.log

# Перезапуск бота если упал
*/10 * * * * systemctl is-active --quiet mybot || systemctl restart mybot
```
